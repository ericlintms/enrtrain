# 設備製造業 IT 工程師 AI 培訓課程大綱

為設備製造業 IT 工程師設計的課程，關鍵在於**「降低底層演算法的篇幅，最大化系統架構、API 部署與資料串接的實作比例」**。這群學員懂系統與資料庫，教材重點應放在如何將 AI 視為一個可調用的微服務（Microservice）。

## Session 1｜地端 LLM 部署與企業系統 API 整合 (1 小時)

**單元目標：** 讓 IT 人員具備在企業內部安全部署開源模型，並將其封裝為 API 供既有 ERP 或機台系統呼叫的能力。

* **10 分鐘：LLM 基礎觀念與邊緣運算優勢**
* 破除迷思：何時該用商業 API (如 OpenAI)，何時必須用 Local LLM（機密保護、斷網可用性）。
* 硬體評估：在工廠端運行模型所需的 VRAM 與算力基礎。


* **20 分鐘：Ollama 部署與 Web API 實戰**
* Ollama 架構解析與安裝實務。
* **教材重點：** 展示如何透過 RESTful API 與 Local LLM 互動。
* 將模型回應整合進企業常用後端語言（例如利用 Python 建立中介層）。


* **20 分鐘：實務應用場景：從文字到視覺 (VLM)**
* **教材重點：** 示範將機台錯誤日誌（Log）或 JSON 格式的工單資料拋給 LLM，讓其輸出結構化判讀結果。
* LLM 讀圖原理：引入 Vision 模型，探討如何辨識設備儀表板或簡單的異常影像。


* **10 分鐘：單元總結與 QA**
* 產出確認：學員能理解如何啟動一個本地端的 AI API Server。



## Session 2｜RAG 私有知識庫：賦予 AI 設備領域知識 (1 小時)

**單元目標：** 拆解 RAG 架構，讓學員理解如何將龐大的「設備操作手冊」與「歷史維修記錄」轉化為 AI 可檢索的資料庫。

* **15 分鐘：RAG 核心運作機制與架構**
* 檢索（Retrieve）、生成（Generate）與向量化（Embedding）的概念轉換。
* 為什麼傳統 SQL 關聯式資料庫查詢不夠？向量資料庫扮演的角色。


* **25 分鐘：企業專屬知識處理實務 (Data Pipeline)**
* **教材重點：** 針對製造業常見的非結構化資料（PDF 說明書、Word 維修報告）進行文本切塊（Chunking）的策略。
* 實作思維：如何處理文件中包含的表格與設備參數。


* **15 分鐘：RAG 進階優化與準確度提升**
* 降低幻覺（Hallucination）的系統性做法。
* 混合搜尋（Hybrid Search）：結合向量語意搜尋與傳統關鍵字搜尋。


* **5 分鐘：單元總結與 QA**
* 產出確認：學員能畫出企業內部建置 RAG 的資料處理與檢索流程圖。



## Session 3｜AI Agent 實戰：打造製程自動化助手 (1 小時)

**單元目標：** 引入 Agent 概念與 MCP 協定，讓 LLM 從「被動回答問題」進化為「主動調用企業工具（如查詢資料庫、發送異常通知）」的輔助程式。

* **15 分鐘：從 LLM 到 AI Agent 的思維躍進**
* 什麼是 Agent？理解提示詞驅動（Prompt-driven）與工具調用（Function Calling / Tool Use）的差異。


* **25 分鐘：LangChain / MCP 架構與企業工具整合**
* 介紹 Model Context Protocol (MCP) 的概念，標準化 AI 存取資料來源的途徑。
* **教材重點：** 模擬一個實務情境——當機台發生 Error Code，Agent 如何透過 SQL 工具自動查詢 MariaDB/資料庫中的歷史維護紀錄，並整理出初步的排查建議。


* **15 分鐘：AI 製程工程師的整合方案探討**
* 將 Session 1 (Local LLM), Session 2 (RAG), 與 Session 3 (Agent) 串聯。
* 探討導入可行性與資安邊界（AI 可以「讀」什麼，有權限「寫/執行」什麼）。


* **5 分鐘：單元總結與 QA**
* 產出確認：學員能針對自身單位的痛點，提出一個具備可行性的 Agent 整合草案。
